四、核心实现思路

async function guardedRunTool(toolCall) {
  const decision = safetyPolicy.evaluate(toolCall);

  if (decision.action === "reject") {
    return { isError: true, content: "Tool blocked: " + decision.reason };
  }

  if (decision.action === "confirm") {
    await askUserToConfirm(toolCall, decision.reason);
  }

  return await runTool(toolCall);
}

五、安全分层

动作默认策略原因
读取项目内普通文本允许低风险，但仍要限制路径。
读取项目外路径 / 密钥拒绝容易泄露隐私和凭据。
写新文档 / 修改源码确认需要让用户知道会改哪里。
运行测试 / lint允许或确认通常低风险，但要限制命令和超时。
