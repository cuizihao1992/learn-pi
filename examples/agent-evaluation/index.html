怎么判断 Agent 是否真的会用工具

Agent 评估不能只看最终回答是否流畅。一个可用 Agent 至少要选对工具、填对参数、执行后把 toolResult 用进最终回答，并且避免多余调用。这个实验把一次 Agent 运行拆成可打分的 trace。

四、评估逻辑

Tool Selection Accuracy:
expectedTools 与 actual tool_calls 的工具名是否匹配。

Argument Accuracy:
工具参数是否满足任务，例如 path 是否是 README.md，command 是否是测试命令。

Tool Result Grounding:
最终回答是否引用或使用了 toolResult 中的关键信息。

Step Efficiency:
是否用尽量少的必要步骤完成任务，是否出现多余工具调用。

Final Answer Correctness:
最终回答是否完成用户任务，并说明观察、修改或验证结果。

五、学习重点

  • Agent 的错误常常不是“模型不会说”，而是工具选择、参数、执行、回填、总结中的某一环断了。
  • 只看 final answer 会掩盖问题，必须保存 toolCall、toolResult、错误和最终回答组成的 trace。
  • 真实项目里要把这些 trace 做成评测集，覆盖读文件、改文件、运行测试、拒绝危险操作等场景。
  • Pi Agent 的源码调试也可以按这个表定位：先看模型有没有 toolCall，再看工具有没有执行，再看 toolResult 有没有回填。
