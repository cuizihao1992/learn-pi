看见一次 RAG 闭环怎么跑完

这个实验用一个小型教学语料和手写 embedding 规则模拟 RAG。它不会调用真实模型，但会把 RAG 的关键中间状态全部摊开：文档如何切片，query 如何向量化，topK 如何召回，证据如何拼进 prompt，回答如何附引用。

一、RAG 流水线

1. Chunk把长文档切成能独立回答问题的知识片段。
2. Embed把问题和 chunk 映射到同一组语义维度。
3. Retrieve用余弦相似度选出 topK 证据。
4. Compose把问题、证据和引用规则拼成 prompt。
5. Answer只根据证据生成答案，并标出来源。

这个 demo 在模拟什么

真实生产 RAG：
files -> parser -> semantic chunks -> embedding model -> vector index
user query -> query rewrite -> embedding -> vector/BM25/hybrid retrieval
topK chunks -> rerank -> context compression -> prompt with citations
LLM -> answer grounded in evidence

这个教学 demo：
固定小语料 -> 手写 chunk -> 6 维教学向量
query vs chunk -> 余弦相似度排序
top2 evidence -> 拼 prompt -> 模拟一个带引用的答案

常见误区

  • 只展示最终答案，看不见中间证据，就无法判断错在召回、排序还是生成。
  • chunk 过小会丢上下文，chunk 过大会把无关内容塞进 prompt。
  • 只用向量检索会漏掉接口名、错误码、股票代码这类精确词。
  • 没有引用规则时，模型很容易把证据外的信息混进答案。
