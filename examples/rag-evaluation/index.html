四、评估逻辑

Retrieval evaluation:
expectedEvidence = 标注的正确证据 ID
retrievedTopK = 系统召回的证据 ID
Recall@K = expectedEvidence 有多少出现在 retrievedTopK 中
MRR = 第一个正确证据排在第几名，越靠前越好

Generation evaluation:
supportedClaims = 答案中能被证据支持的关键断言
unsupportedClaims = 答案中证据没有支持的断言
missingClaims = 标准答案要求但生成答案遗漏的断言
faithfulness = supported / (supported + unsupported)
completeness = coveredExpectedClaims / expectedClaims

五、学习重点

  • 检索没召回正确证据时，生成模型再强也容易答错。
  • 检索召回了正确证据，答案仍可能不忠实，比如混入常识、过度推断或漏掉关键条件。
  • RAG 评估集要保存问题、期望证据、标准要点、坏答案案例和人工判定原因。
  • 线上优化时要能定位问题来自 parsing、chunk、embedding、topK、rerank、prompt 还是模型。
