Part 9

动手练习：从会读到会改

真正理解 agent 运行方式，最好做几个小改动。下面每个练习都尽量小，但会迫使你穿过关键架构边界。

练习 1：画出一次 prompt 调用图

从 `main.ts` 开始，追到 `AgentSession.prompt()`、`Agent.run()`、`runLoop()`。输出一张调用图。

练习 2：新增一个只读工具

新增 `stats` 工具，返回某个文件的行数、字节数、后缀名。模仿 `ls.ts` 或 `grep.ts`，注册到 `tools/index.ts`。

练习 3：观察 toolResult 如何驱动下一轮

在测试里构造一个模型先调用 read，再根据 toolResult 返回最终回答。目标是理解 agent loop 为什么会多轮运行。

练习 4：降低压缩阈值

让 compaction 更容易触发，观察 summary message 插入位置。记录哪些历史被压缩，哪些近期消息保留。

练习 5：给工具执行 UI 增加一个字段

例如在 bash 工具展示里显示 timeout。这样你会同时读工具定义、工具事件和 interactive component。

练习 6：写一篇自己的架构解释

用这句话开头：Pi Agent 运行一次任务时，模型只负责决定下一步，真正的执行和状态管理由 Pi 负责。然后自己补全 500 字。

完成标准你能不看源码说清楚四层架构和一次 toolCall 流程
