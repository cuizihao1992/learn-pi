Today I Learned

把零散学习变成可搜索资产

TIL 不是正式教程，而是知识进入系统的入口。每次学到一个概念、一个工程坑、一个论文结论或一个现实案例，先用短条目记录下来；成熟后再迁移到对应模块。

一、TIL 条目模板

标题：一句话说明今天学到什么
日期：YYYY-MM-DD
模块：LLM / Agent / Map / Finance / Rendering / Science
一句话模型：这个知识点解释什么
例子：一个最小例子或现实场景
关联概念：3 到 5 个相关术语
下一步：要补到哪个正式页面

二、推荐记录什么

类型例子成熟后去哪里
概念Embedding 是语义坐标，RAG 用它找资料。概念索引 / Embedding 与多模态
工程坑RAG chunk 只按长度切会切断标题和正文。RAG 数据工程
代码观察Agent Loop 必须把 toolResult 回填给模型。Mini Agent
现实案例降息不一定利好股市，要同时看折现率、盈利和预期。经济金融案例
系统类比DrawCommand 像渲染系统的最终执行指令。Cesium 渲染路径

三、本周种子条目

条目一句话模型关联
Embedding把文本、图片、代码等对象变成可比较的向量。Embedding 与多模态
RAG先检索外部证据，再让 LLM 基于证据回答。RAG 数据工程
多模态模型把图像、音频、视频等模态对齐到语言模型可用的表示空间。Embedding 与多模态
系统课程一个模块要有总览、路线、概念、案例、练习和验收。经济金融系统课程

四、沉淀规则

  1. 先短后长：先写 100 字，不等完美。
  2. 先链接后扩写：至少连到一个模块和三个概念。
  3. 每周整理一次：把成熟条目迁移到正式页面。
  4. 每月清理一次：把重复条目合并，把过时条目标注边界。
