新增内容时怎么写

这个网站后续会继续长大，所以每次新增模块或章节都要有固定结构：先定义学习目标，再画完整链路，再解释关键抽象，最后给练习、测验、速查和下一步。

新增模块必须回答

  • 这个模块解决什么学习目标？
  • 它和已有模块是什么关系？
  • 小白学完能画出哪条链路？
  • 最小可运行 demo 是什么？
  • 容易混淆的概念有哪些？
  • 如何自测是否真的理解？

推荐目录

modules/new-topic/
  index.html          # 模块总览
  charter.html        # 模块宪章
  architecture.html   # 架构或核心模型
  lifecycle.html      # 完整运行链路
  debugging.html      # 调试手册
  glossary.html       # 术语表
  labs.html           # 动手练习

单页写作结构

  1. 一句话说明这页学什么。
  2. 用流程图或代码块画完整链路。
  3. 把核心抽象拆成表格。
  4. 解释为什么这样设计，而不是只描述是什么。
  5. 列出常见错误和调试方法。
  6. 给一个最小练习或验收标准。
  7. 链接到相关模块，形成学习网络。

页面质量检查

检查项通过标准
小白友好第一屏能看出为什么要学。
链路完整能从输入走到输出，不只讲局部名词。
对比清楚容易混淆的概念必须放表格。
能动手至少有一个练习、任务或验收标准。
能检索新增页面后重建 search-index.json。
不断链跑链接扫描，所有相对链接存在。

新增后必须更新

  • content-manifest.json：登记模块或公共页面。
  • README.md：更新目录说明。
  • pages/learning-paths.html：如果影响学习路线，要接进去。
  • pages/concept-index.html：如果有新术语，要加入索引。
  • search-index.json：运行构建脚本重建。
