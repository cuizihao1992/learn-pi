Module Charter

Cesium 模块宪章

本模块目标是让读者理解三维地球引擎如何把业务对象、瓦片、地形、影像和 3D Tiles 转换成 WebGL 绘制。

完成标准

  • 能说清 Viewer、Scene、Primitive、DrawCommand、Renderer 的职责。
  • 能画出 `Entity -> Visualizer -> Primitive -> DrawCommand -> WebGL`。
  • 能解释 Globe、Terrain、Imagery、3D Tiles 为什么有特殊路径。
  • 能按渲染链路定位“为什么没画出来”。

核心链路

Viewer
  -> CesiumWidget
  -> Scene.render()
  -> frameState
  -> Primitive/Globe/Tileset.update()
  -> DrawCommand
  -> Renderer
  -> WebGL
  -> GPU pixels

源码入口

  • `packages/engine/Source/Widgets/Viewer/Viewer.js`
  • `packages/engine/Source/Scene/Scene.js`
  • `packages/engine/Source/Scene/Primitive.js`
  • `packages/engine/Source/Renderer/DrawCommand.js`
  • `packages/engine/Source/Renderer/Context.js`
  • `packages/engine/Source/Scene/Globe.js`
  • `packages/engine/Source/Scene/Cesium3DTileset.js`

模块内容标准

  • 任何渲染章节都必须说明对象如何进入 commandList。
  • 必须区分 Entity 和 Primitive。
  • 必须解释 pass、RenderState、ShaderProgram、VertexArray。
  • 调试内容必须沿 Viewer -> Scene -> Command -> Renderer 查。
