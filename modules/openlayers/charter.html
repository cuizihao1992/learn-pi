Module Charter

OpenLayers 模块宪章

本模块目标是让读者理解二维 Web 地图如何从 Map、View、Layer、Source 走到 Canvas/WebGL renderer。

完成标准

  • 能解释 Map、View、Layer、Source 的职责。
  • 能区分 TileLayer、ImageLayer、VectorLayer、VectorTileLayer。
  • 能说明 Layer 和 Source 为什么分离。
  • 能按 target、View、Layer、Source、renderer 顺序调试地图空白。

核心链路

new Map({ target, view, layers })
  -> View state
  -> LayerGroup
  -> LayerRenderer
  -> Source loads tiles/features/images
  -> frameState/layerState
  -> Canvas or WebGL
  -> browser pixels

源码入口

  • `src/ol/Map.js`
  • `src/ol/View.js`
  • `src/ol/layer/Layer.js`
  • `src/ol/source/Source.js`
  • `src/ol/renderer/`
  • `src/ol/layer/Tile.js`、`Vector.js`、`VectorTile.js`

模块内容标准

  • 任何章节都要说明 View state 如何影响数据选择。
  • Layer 章节必须同时讲 Source。
  • 渲染章节必须区分 Canvas 和 WebGL。
  • 调试章节必须覆盖投影和坐标转换。
