Module Charter

Mini Agent 模块宪章

这个模块不是继续讲 Pi 源码，而是让小白能从零写出自己的最小 Agent。它负责把“Agent 的运行原理”落到可运行代码、接口兼容和产品化扩展上。

学习目标

  • 理解最小 Agent Loop：模型决策、工具执行、结果回填。
  • 能用 Node.js 跑通一个 OpenAI-compatible Mini Agent。
  • 知道自己的模型如何通过中间层兼容 OpenAI/Anthropic 风格接口。
  • 知道 MVP 到可用产品还缺哪些工程能力。

模块边界

Pi Agent 模块研究现成开源项目 Pi 的架构；Mini Agent 模块研究如何自己实现一个最小 Agent。两个模块互相参照，但不混在一个目录里。

必须覆盖的内容

  1. MVP 代码必须能被复制和运行。
  2. 每个工具调用都要讲清楚请求格式、执行位置、回填格式。
  3. 模型接口章节必须区分“训练能力”和“中间层协议转换”。
  4. 产品化路线必须覆盖权限、会话、压缩、流式事件、错误恢复。
