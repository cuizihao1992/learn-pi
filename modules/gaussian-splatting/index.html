Gaussian Splatting

高斯泼溅：从照片到实时新视角渲染

3D Gaussian Splatting 的直觉是：不用把场景存成网格，也不用像 NeRF 那样每条光线采样很多点，而是把场景表示成大量可优化的 3D 高斯椭球。渲染时把它们投影到屏幕、排序、混合，就能实时生成新视角。

它解决什么问题

输入

一组照片或视频帧、相机内外参、稀疏点云。通常先用 COLMAP 做相机标定和稀疏重建。

输出

一组 3D Gaussian：位置、尺度、旋转、颜色/球谐系数、不透明度。它们共同表示一个可实时渲染的场景。

训练

通过可微 rasterizer 渲染当前视角，与真实照片计算损失，反向优化每个 Gaussian 的参数。

实时渲染

把高斯投影成屏幕空间椭圆，按深度排序后 alpha blending，避免 NeRF 那种重采样开销。

先记住这条链路

多视角照片
  -> COLMAP 求相机位姿和稀疏点云
  -> 初始化 3D Gaussians
  -> 可微 rasterization 渲染训练视角
  -> 和真实图片算 loss
  -> 优化位置、协方差、颜色、不透明度
  -> densify / prune 调整高斯数量
  -> 实时 viewer 投影、排序、混合

和其他模块的关系

  • 和 Cesium：Cesium 讲引擎如何调度 DrawCommand；高斯泼溅讲场景本身如何被表示为很多透明椭球。
  • 和 OpenLayers：OpenLayers 是二维地图数据到 Canvas/WebGL；高斯泼溅是三维视觉重建数据到屏幕像素。
  • 和 Mini Agent：Mini Agent 是决策和工具闭环；高斯泼溅是训练和渲染闭环。

推荐学习顺序

  1. 先理解 3D Gaussian 的参数：mean、covariance、opacity、color/SH。
  2. 理解 COLMAP 为什么提供初始化相机和点云。
  3. 理解训练循环：render -> loss -> backward -> optimizer。
  4. 理解 densification 和 pruning 为什么影响质量与显存。
  5. 理解实时渲染：projection、tile/binning、depth sort、alpha blend。
