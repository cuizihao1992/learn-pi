Render Path

高斯泼溅完整路径：从相机到像素

高斯泼溅要分成两条链路看：训练时的可微渲染链路，以及 viewer 中的实时渲染链路。两者都依赖同一组 Gaussian 参数，但目标不同。

1. 数据准备

images/
  -> COLMAP feature extraction
  -> feature matching
  -> sparse reconstruction
  -> cameras + images + sparse points

COLMAP 给出相机内参、外参和稀疏点云。稀疏点云不是最终渲染结果，而是初始化 Gaussian 位置的起点。

2. Gaussian 参数

参数作用为什么重要
mean / xyz高斯中心点决定场景几何位置。
scale + rotation构造 3D 协方差决定高斯椭球大小和方向。
opacity透明度决定 alpha blending 贡献。
color / SH颜色或视角相关颜色决定照片级外观。

3. 训练链路

取一个训练相机
  -> project 3D Gaussian 到屏幕空间
  -> 计算屏幕椭圆覆盖哪些 tile / pixel
  -> 按深度组织可见贡献
  -> alpha blending 得到渲染图
  -> 和真实照片算 L1 / SSIM loss
  -> 反向传播更新 Gaussian 参数
  -> densify / prune 控制点数

训练的核心不是“画出来”这么简单，而是让 rasterizer 对 Gaussian 参数可微。只有可微，loss 才能告诉每个高斯该移动、变大、变小、变透明还是变颜色。

4. 3D 协方差到 2D 椭圆

每个 Gaussian 在 3D 里是一个椭球。渲染时要通过相机投影和雅可比近似，把 3D 协方差变成屏幕空间的 2D 协方差。直觉上就是：一个空间椭球从当前相机看过去，会覆盖屏幕上的一个椭圆区域。

3D mean + covariance
  -> view transform
  -> projection jacobian
  -> 2D covariance
  -> screen-space ellipse
  -> tile coverage

5. 为什么要排序和混合

高斯是半透明贡献，不像普通 mesh 那样只做最近深度测试就完事。多个高斯覆盖同一个像素时，需要按深度从前到后或从后到前合成，使用 alpha blending 累积颜色和透明度。

pixel color = sum(transmittance * alpha_i * color_i)
transmittance *= (1 - alpha_i)

6. densification / pruning

  • densification：某些区域误差大或梯度大，说明需要更多高斯表达细节，于是 clone/split。
  • pruning：透明度太低、贡献太小或异常的高斯会被删除，控制显存和噪点。
  • reset opacity：训练中可能重置透明度，让优化摆脱局部坏状态。

7. 和 NeRF 的关键区别

点NeRF3D Gaussian Splatting
场景表示神经网络隐式场显式 3D Gaussians
