Part 5

会话、分支和上下文压缩

Agent 不只是“本轮回答”。它要记住之前读过什么、改过什么、工具结果是什么，还要在上下文太长时继续工作。

会话保存什么

  • 用户消息、助手消息、工具调用结果。
  • bash 执行记录、custom message、summary message。
  • token usage、cost、时间戳、父子关系。

为什么是树，不只是列表

因为用户可能从旧会话分叉继续。树结构能支持 fork、branch summary、切换会话路径。线性列表适合简单聊天，但不适合 coding agent 的探索和回退。

上下文为什么需要压缩

模型上下文窗口有限。长任务里工具输出很多，如果全部塞回模型，很快会超限。压缩就是把旧 turn 总结成一条 summary message。

完整历史太长
  -> estimateContextTokens()
  -> shouldCompact()
  -> findCutPoint()
  -> generateSummary()
  -> compact()
  -> summary message + 最近消息继续进入模型

好的压缩摘要要保留什么

  • 读过和改过的文件路径。
  • 关键函数名、类名、配置项。
  • 错误信息和失败命令。
  • 已经做完的决策和还没完成的任务。
