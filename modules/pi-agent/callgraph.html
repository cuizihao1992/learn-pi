源码调用图：从入口追到工具执行

这一页给你“读代码路线”。不知道从哪里跳到哪里时，就按这些图走。

主路径：一次用户输入

packages/coding-agent/src/main.ts
  -> createAgentSessionRuntime()
  -> createAgentSessionServices()
  -> createAgentSessionFromServices()
  -> runInteractiveMode() / runPrintMode() / runRpcMode()
  -> AgentSession.prompt()
  -> Agent.run()
  -> runAgentLoop()
  -> runLoop()
  -> streamAssistantResponse()
  -> pi-ai streamSimple()

工具路径：模型返回 toolCall 后

runLoop()
  -> message.content.filter(type === "toolCall")
  -> executeToolCalls()
  -> executeToolCallsParallel() or executeToolCallsSequential()
  -> prepareToolCall()
  -> validateToolArguments()
  -> executePreparedToolCall()
  -> finalizeExecutedToolCall()
  -> createToolResultMessage()
  -> currentContext.messages.push(toolResult)

工具注册路径

AgentSession._buildRuntime()
  -> createAllToolDefinitions()
  -> createToolDefinition(read/bash/edit/write/grep/find/ls)
  -> wrapRegisteredTools()
  -> _toolDefinitions / _toolRegistry
  -> Agent.state.tools

前端事件路径

runLoop emit AgentEvent
  -> Agent.subscribe(listener)
  -> AgentSession._handleAgentEvent()
  -> AgentSessionEvent
  -> interactive-mode listener
  -> components update
  -> pi-tui render

压缩路径

AgentSession._handlePostAgentRun()
  -> estimateContextTokens()
  -> shouldCompact()
  -> prepareCompaction()
  -> generateSummary()
  -> compact()
  -> session entries include summary message

参考答案：练习 1 调用图

如果你做了 labs 里的“画出一次 prompt 调用图”，参考答案可以是：

用户输入
  -> interactive-mode 捕获输入
  -> AgentSession.prompt(text)
  -> create user AgentMessage
  -> Agent.run([message])
  -> runAgentLoop()
  -> runLoop()
  -> streamAssistantResponse()
  -> pi-ai provider stream
  -> assistant message with toolCall
  -> executeToolCalls()
  -> toolResult
  -> runLoop next turn
  -> assistant final text
  -> agent_end
