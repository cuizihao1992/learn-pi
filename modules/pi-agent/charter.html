Module Charter

Pi Agent 模块宪章

本模块目标是让读者理解一个 coding agent 如何从用户提示词出发，调用模型、执行工具、保存会话、压缩上下文，并扩展到自研模型接入。

目标读者

  • 想知道 agent 到底怎么运行的新手。
  • 想读 Pi 源码的工程师。
  • 想开发自己 agent 或接入自研模型的人。

完成标准

  • 能画出 `prompt -> model -> toolCall -> toolResult -> final answer` 链路。
  • 能解释 Agent、AgentSession、runLoop 的区别。
  • 能写出一个 Node Mini Agent MVP。
  • 能说明自研模型为什么通常需要 OpenAI-compatible adapter。

核心链路

用户 prompt
  -> AgentSession.prompt()
  -> Agent.run()
  -> runLoop()
  -> pi-ai streamSimple()
  -> assistant toolCall
  -> executeToolCalls()
  -> ToolResultMessage
  -> 下一轮模型调用
  -> AgentEvent 返回前端

源码入口

  • `packages/coding-agent/src/main.ts`
  • `packages/coding-agent/src/core/agent-session.ts`
  • `packages/agent/src/agent.ts`
  • `packages/agent/src/agent-loop.ts`
  • `packages/coding-agent/src/core/tools/index.ts`
  • `packages/coding-agent/src/core/compaction/compaction.ts`

模块内容标准

  • 任何新章节都必须包含一个具体对话例子。
  • 工具章节必须说明 toolCall 和 toolResult 的 JSON 形状。
  • 模型章节必须区分“模型能力”和“API adapter”。
  • 调试章节必须按提示词、模型、工具、会话、前端分层定位。
